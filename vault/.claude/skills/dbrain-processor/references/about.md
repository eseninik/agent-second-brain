## Task Management Preferences

### Task Title Style
**Hybrid approach:** Короткий заголовок + контекст в description

✅ Good examples:
- Title: "Отправить презентацию собственнику"
  Description: "Презентация бот-квалификатора для утверждения концепции"

- Title: "Созвон с Павлом по MVP"
  Description: "Показать первое MVP проекта для отдела исполнения (пятница)"

❌ Bad examples:
- "Подумать о боте" (слишком размыто)
- "Презентация бот-квалификатора для утверждения концепции с собственником для согласования дальнейшей разработки и внедрения" (слишком длинно)

### Books Tracking
- Добавлять книги как задачи в TickTick (проект "Learning")
- Можно использовать подзадачи для глав
- Цель: 2-3 книги в месяц → 30+ за год

### Health & Habits Tracking
Отслеживать через TickTick (проект "Personal"):
- **Тренировки на площадке:** ежедневные задачи, прогресс в подтягиваниях (цель: 7+ раз)
- **Чтение книг:** прогресс по книгам

## Daily Assistant Preferences

### Report Timing
- **Автоматический отчёт:** каждый день в 21:00
- **Ручной запуск:** доступен по команде /process в любое время

### Report Content (Must Include)
1. **Прогресс по целям** — как сегодняшние задачи связаны с месячными/годовыми целями
2. **Загрузка на неделю** — сколько задач на каждый день
3. **Просроченные задачи** — предупреждения о пропущенных дедлайнах
4. **Топ-3 приоритета на завтра** — что самое важное сделать завтра

### Goal Stale Alerts
- Показывать КАЖДЫЙ ДЕНЬ, если цель без активности 7+ дней
- Формат: "⚠️ Цель '{goal}' без активности {days} дней"

### Key Assistant Priorities
1. **Не пропустить важное** — дедлайны и критичные задачи
2. **Экономить время** — минимум ручной работы
3. **Структурировать мысли** — хаос → action items
4. **Видеть прогресс к целям** — связь ежедневных задач с большими целями

## Anti-Patterns (AVOID)

При обработке записей НЕ делать:
- Создавать абстрактные заметки без action items
- Делать задачи слишком размытыми («подумать о...»)
- Дублировать контент без синтеза
- Создавать списки без приоритетов
- Игнорировать дедлайны и просроченные задачи
- Создавать слишком длинные заголовки задач (используй description)
